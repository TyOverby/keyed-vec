IndexedVec is a vector with a unique approach to indices. Once an item is added to the IndexedVec, a unique index is returned. This index is the only way that that element may be accessed, and it is garunteed by the rust typesystem for there only to be one of these indices at one point in time.

This means that we can perform operations that would otherwise be unsafe in a perfectly safe maner. For example, you can grab a mutable reference to an element from an immutable IndexedVec. This is hugely useful in multithreaded environments.

Because the Index is garunteed to exist and point to a valid location in the backing array, the implementation of IndexedVec can also do lookups without bounds checking.
